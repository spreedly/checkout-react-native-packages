# Source Maps Security Guide - React Native SDK

**Version:** 1.0  
**Last Updated:** November 2025  
**Purpose:** Security best practices for source map handling in React Native SDK

## Overview

Source maps are files that map minified/compiled code back to original source code. In React Native, source maps are generated for:

- **JavaScript bundles** (via Metro bundler)
- **Hermes bytecode** (Android - via Hermes compiler)
- **Native code** (Android/iOS - via build tools)

**Security Risk:** If source maps are distributed with production apps, they can expose:

- Original source code structure
- Business logic
- API endpoints
- Internal implementation details
- Sensitive comments or code

## Security Checklist

- [ ] ✅ Source maps are **generated** for debugging purposes
- [ ] ✅ Source maps are **NOT distributed** in production builds
- [ ] ✅ Source maps are **stored separately** for internal debugging
- [ ] ✅ Source maps are **disabled** in production release builds
- [ ] ✅ Source maps are **excluded** from app bundles/APKs
- [ ] ✅ Source maps are **withheld** from public distribution

## Implementation Guide

### 1. Android Configuration

#### Disable Hermes Source Maps in Production

Edit `example/android/app/build.gradle`:

```gradle
react {
    /* Hermes Commands */
    // Disable source map generation in release builds
    hermesFlags = [
        "-O",
        // Remove "-output-source-map" for production
        // Only include for debug: "-output-source-map"
    ]

    // Or conditionally disable based on build type
    if (buildType == "release") {
        hermesFlags = ["-O"]  // No source map
    } else {
        hermesFlags = ["-O", "-output-source-map"]  // Include for debug
    }
}
```

#### Exclude Source Maps from APK

Add to `android` block in `build.gradle`:

```gradle
android {
    buildTypes {
        release {
            // Exclude source maps from release APK
            packagingOptions {
                exclude "**/*.map"
                exclude "**/*.sourcemap"
                exclude "**/index.android.bundle.map"
            }
        }
    }
}
```

### 2. iOS Configuration

#### Disable Source Maps in Xcode

1. **Xcode Build Settings:**
   - Navigate to: Build Settings → Swift Compiler - Code Generation
   - Set **Debug Information Format** to:
     - **Debug**: `DWARF with dSYM File` (includes source maps)
     - **Release**: `DWARF` (no source maps)

2. **React Native Metro Bundle:**
   - Source maps are generated by Metro bundler
   - Ensure they're not included in the app bundle

#### Exclude Source Maps from App Bundle

In Xcode:

1. Go to Build Phases → Copy Bundle Resources
2. Remove any `.map` or `.sourcemap` files
3. Ensure they're not copied to the app bundle

### 3. Metro Bundler Configuration

#### Disable Source Maps in Production

Edit `example/metro.config.js`:

```javascript
const path = require('path');
const { getDefaultConfig } = require('@react-native/metro-config');
const { withMetroConfig } = require('react-native-monorepo-config');

const root = path.resolve(__dirname, '..');

module.exports = withMetroConfig(getDefaultConfig(__dirname), {
  root,
  dirname: __dirname,
  // Disable source maps in production
  transformer: {
    getTransformOptions: async () => ({
      transform: {
        experimentalImportSupport: false,
        inlineRequires: true,
      },
      // Disable source maps for production builds
      sourceMapUrl:
        process.env.NODE_ENV === 'production' ? undefined : 'inline',
    }),
  },
});
```

#### Environment-Based Configuration

Create separate build scripts:

**package.json:**

```json
{
  "scripts": {
    "bundle:android:release": "react-native bundle --platform android --dev false --entry-file index.js --bundle-output android/app/src/main/assets/index.android.bundle --assets-dest android/app/src/main/res --sourcemap-output android/app/src/main/assets/index.android.bundle.map",
    "bundle:android:production": "react-native bundle --platform android --dev false --entry-file index.js --bundle-output android/app/src/main/assets/index.android.bundle --assets-dest android/app/src/main/res",
    "bundle:ios:release": "react-native bundle --platform ios --dev false --entry-file index.js --bundle-output ios/main.jsbundle --assets-dest ios --sourcemap-output ios/main.jsbundle.map",
    "bundle:ios:production": "react-native bundle --platform ios --dev false --entry-file index.js --bundle-output ios/main.jsbundle --assets-dest ios"
  }
}
```

**Note:** Use `bundle:android:production` and `bundle:ios:production` for production builds (no `--sourcemap-output` flag).

### 4. Store Source Maps Separately

#### For Internal Debugging Only

1. **Generate source maps separately:**

   ```bash
   # Generate source map but don't include in bundle
   react-native bundle \
     --platform android \
     --dev false \
     --entry-file index.js \
     --bundle-output android/app/src/main/assets/index.android.bundle \
     --sourcemap-output ./sourcemaps/android-release.map
   ```

2. **Store in secure location:**
   - Store source maps in version control (private repository)
   - Or use a secure artifact storage system
   - Never include in public releases or app stores

3. **Access control:**
   - Only accessible to internal development team
   - Not included in customer-facing distributions
   - Separate from production builds

## Troubleshooting

### Source Maps Still Appearing in Production Build

**Check:**

1. Build configuration (Gradle/Xcode)
2. Metro bundler settings
3. Hermes compiler flags
4. Packaging options (exclude `.map` files)
5. Bundle generation scripts

### Source Maps Not Generating for Debugging

**Solution:**

- Enable source maps only in debug builds
- Use separate build scripts for debug vs production
- Configure environment variables to control source map generation
